USER FLOW (HOW APP WORKS)

1. User opens app
- Can browse movies and TV (from DB + TMDB)

2. When user opens a movie/TV:
- System checks DB first
- If not present → fetch from TMDB → store in DB
- Then return data

3. TV Flow:
- TV → seasons → episodes
- Each episode has its own streaming links

4. Streaming (CRITICAL FLOW)

User clicks Play:

/watch/[id]?type=movie
OR
/watch/[id]?type=episode

Backend:

GET /api/links/[id]

Logic:

- Fetch links from DB
- For each link:

IF type = "direct":
  return url directly

IF type = "extract":
  IF extracted_url exists AND not expired:
    return extracted_url
  ELSE:
    call extractor (Cloudflare Worker or API)
    get real video URL
    store extracted_url + expires_at (now + 6 hours)
    return it

5. If no links:
- Return null
- UI shows "Not Available"

6. Language Filtering:

- Based on links.languages JSON
- Movie appears ONLY ONCE even if multiple qualities match
