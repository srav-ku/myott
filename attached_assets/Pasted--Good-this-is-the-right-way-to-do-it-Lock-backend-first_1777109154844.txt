# 2. CACHING — what you didn’t fully get

You asked:

> what is cached response and how it avoids TMDB calls?

---

## Simple explanation:

Think like this:

### First user:

- opens Batman
    
- your backend calls TMDB
    
- gets recommendations
    
- **stores response in cache (temporary memory)**
    

---

### Next user:

- opens Batman
    
- instead of calling TMDB  
    👉 it returns **saved cached data**
    

---

## Code example:

```ts
fetch(url, { next: { revalidate: 86400 } }) // 1 day
```

---

## Meaning:

- Cache this response for 1 day
    
- Don’t hit TMDB again during that time
    

---

## So result:

|User|TMDB Call|
|---|---|
|1st user|YES|
|next 100 users|NO|

---

👉 That’s how it saves API usage
