complete backend blueprint in simple terms.

🎯 WHAT THIS APPLICATION IS

This is an OTT-style platform (like Prime Video UI) with:

Normal browsing (movies & TV)
Hidden streaming system (admin-controlled)
Manual + TMDB-based content
Fully controlled backend (no auto scraping chaos)
🧱 TECH STACK (FULL CONTEXT)

This system is built using:

Frontend (later connection)
Next.js (App Router)
React
Tailwind CSS
Backend
Next.js API Routes (main logic)
Cloudflare Workers (optional → for extraction)
Database
Cloudflare D1 (SQLite)
ORM
Drizzle ORM
External API
TMDB (for metadata like posters, overview, recommendations)
🧠 HOW SYSTEM WORKS (BIG PICTURE)

There are 3 main flows:

User Flow (what users see)
Admin Flow (how content is controlled)
Technical Flow (how backend works internally)
👤 1. USER FLOW (SIMPLE VIEW)
When user opens app:

They can:

Browse movies / TV shows
Search content
Open movie or series page
Movie page shows:
Poster, title, rating
Overview
Genres
Languages (based on links)
Recommendations (from TMDB)
TV show page shows:
Seasons
Episodes list
Each episode has Play button
When user clicks Play:

System goes to:

/watch/[id]?type=movie
OR
/watch/[id]?type=episode
What happens:
Backend finds streaming links
Picks correct link (1080p / 720p)
If direct → opens immediately
If extract → resolves real stream first
If no link:

User sees:

👉 “Not available”

If link broken:

User can:

👉 Report issue

Watchlist:
User can save movie/TV
Stored in DB
Shown in “Watchlist” page
🛠️ 2. ADMIN FLOW (VERY IMPORTANT)

Admin controls everything manually.

Admin can:
1. Add content
Add Movie manually
Add TV show manually
Add Episodes (season-wise)

OR

Import from TMDB (optional)
2. Add streaming links

For:

Movie → add 1080p / 720p
Episode → add 1080p / 720p

Each link has:

type → direct / extract
url
languages
3. Manage languages

Admin can:

Add new language (like Korean)
No code change needed
4. Fix issues

Admin sees:

Broken link reports
No link reports

Then:

Updates link in DB
5. View missing content

From:

search_logs
content_requests

Admin knows what users want

6. Bulk operations
Add episodes in one form
Upload CSV (season data)
⚙️ 3. TECHNICAL FLOW (CORE LOGIC)

This is the most important part.

🔗 HOW LINKS SYSTEM WORKS

This is the core of streaming.

Links table structure:

Each row = ONE link

Example:

Movie ID = 10
Quality = 1080p
Type = direct
URL = actual link
Languages = ["en","te"]
IMPORTANT RULES
1 row per quality
Movie OR Episode (not both)
Languages stored as JSON
🎬 STREAMING FLOW (STEP BY STEP)
Step 1: User clicks play

Calls:

GET /api/links/[id]?type=movie/episode
Step 2: Backend fetches links

From DB:

All links for that movie/episode
Step 3: Handle each link
Case A: direct

Return:

url
Case B: extract

Check:

If extracted_url exists AND not expired → use it

Else:

Call extraction service (worker)
Get real video URL
Save:
extracted_url
expires_at

Return extracted_url

🔥 EXTRACTION SYSTEM

Extraction means:

Input = webpage link
Output = real video file (.mp4 / .m3u8)
Where it runs:

Preferred:

👉 Cloudflare Worker

Flow:

Next.js API → Worker → returns stream → save in DB
Expiry system:
expires_at = now + 6 hours

Avoids repeated extraction.

🌐 TMDB FLOW

Used for metadata only.

When user opens movie:
Check DB
If not exists → fetch from TMDB → save
If exists → use DB
Recommendations:

Always fetched from TMDB

BUT:

✔ Cached by platform
✔ Not called every time

🧠 CACHE SYSTEM (IMPORTANT)

You do NOT store cache manually.

Platform handles it:

Next.js caching
Cloudflare CDN
Result:
First user → API call
Next users → cached response
🌍 LANGUAGE SYSTEM

Languages are NOT hardcoded.

Stored in:

languages table

Example:

en → English
hi → Hindi
te → Telugu
ko → Korean
Links store:
["en","te"]
Filtering:

When user clicks Telugu:

WHERE languages CONTAINS "te"
Important:

Even if:

1080p = Telugu
720p = Telugu

👉 Movie shows ONLY ONCE

🎯 GENRE SYSTEM

Stored as JSON in movies/tv:

["Action","Drama"]

Filtering:

Simple contains match

No need separate table now

→ no recommendations
